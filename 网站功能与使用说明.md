## 1. 网站整体用途

这个网站把一项无人机任务从“任务产生”到“飞行、机载推理、现场作业、结果回传、返航补能”的过程拆开显示，使非程序开发人员也能回答以下问题：

- 当前展示的是哪一种任务场景？
- 数据来自实时浏览器代理、Python仿真、外部日志，还是实验数据包？
- 每架无人机现在在哪里、飞多高、速度多快、电量和温度如何？
- 无人机为什么执行当前动作？
- 通信中断、低电量、高温、多机冲突和禁飞区如何影响任务？
- 一组实验的20个固定 seed 统计结果如何变化？
- 一个具体代表 seed 中，真实事件和航迹是怎样发生的？
- 基准案例与压力案例在同一时间尺度下有什么区别？

网站内置三类任务场景：

| 场景 | 核心任务 | 主要关注点 |
|---|---|---|
| 城市物资配送 | 从保障站领取药品、样本、备件等物资并送达指定地点 | 截止时间、楼宇环境、能量、签收结果回传 |
| 灾后搜索救援 | 搜索人员、复核热源、评估道路和安置点 | 通信不稳定、高优先级任务、热负载、结果缓存 |
| 农田精准喷洒 | 按风险和优先级完成作业条带喷洒 | 药量、电量、温度、覆盖效率和地形 |

### 1.1 网站不能代表什么

- 不是实际飞控地面站，不会向真实无人机发送控制指令。
- 默认“动力学回放”是平滑轨迹代理，不是RotorPy或Gym-PyBullet-Drones的真实运行结果。
- 场景中的建筑、道路、农田、瓦砾、丘陵等“场景代理”用于帮助理解空间，不是实验计算输入。
- Non-IID联邦曲线是数学关系演示，不是课题最终训练结果。
- 未导入真实`benchmark.json`时，参数量、推理延迟和峰值内存不会填入虚构数值。

## 2. 开始使用与推荐阅读顺序

### 2.1 在线访问或离线打开

在线访问：

<https://yzl-dragon.github.io/uav-mission-glassbox/>

在线版由 GitHub Actions 在自动测试通过后部署。它仍是纯浏览器单文件页面，不依赖仓库子路径中的脚本、样式、图片或数据资源。

需要断网使用、内部演示或单独传递文件时，双击项目目录中的：

```text
打开无人机可视化网站.html
```

推荐使用较新版本的Chrome、Edge或Safari。网站所有主要代码和内置数据都已经嵌入HTML，因此可以断网使用。

自行导入的外部飞行日志和`benchmark.json`只在当前浏览器内读取，不会上传到GitHub服务器。内置数据为公开仿真数据；网站不连接、不指挥、不控制真实无人机。公开仓库当前没有附带开源许可证，公开访问不等于授权复制、修改或再发布源码。

### 2.2 第一次观看的推荐顺序

1. 在页面顶部选择“物资配送”“搜索救援”或“农田喷洒”。
2. 保持“实时代理”，观察任务从接收、飞行到回传和返航的完整链条。
3. 点击地图上的任意无人机，查看右侧逐机遥测。
4. 打开地图上的通信、电量、温度或多机冲突图层，理解专属信息。
5. 切换到“Python回放”，用底部时间轴逐事件检查原始仿真数据。
6. 向下滚动到“五组实验工作台”，阅读20-seed均值、95%置信区间和热图。
7. 在实验工作台中选择一个代表案例，点击“在主地图回放此案例”。
8. 继续查看“阶段4C”，在双地图中对照基准与压力案例。

### 2.3 页面大致阅读顺序

页面从上到下依次为：

## 3. 首先分清数据来源

这是正确理解网站的第一步。页面中同时存在真实源数据、浏览器代理数据、统计结果和解释性背景，不应混为一类。

| 数据类型 | 页面名称 | 是否可改 | 主要用途 | 应如何解读 |
|---|---|---:|---|---|
| 浏览器状态机 | 实时代理 | 是 | 交互式演示任务链、机群规模、任务下发和防卡死避障 | 是网站实时计算的科研代理，不是真实飞行 |
| Python原始结果 | Python回放 | 否 | 按`timeline.csv`逐事件回放，并读取`summary.json` | 位置、动作和汇总均来自对应Python数据 |
| 平滑回放 | 动力学回放 | 否 | 对Python时间线进行二阶平滑插值 | 轨迹更连续，但不冒充真实物理引擎 |
| 用户日志 | 外部日志 | 否 | 回放RotorPy、PyBullet、PX4等转换后的标准JSON | 只显示文件实际提供的字段，不补旧指标 |
| 多次运行统计 | 实验工作台 | 否 | 展示20个固定seed的均值、95%置信区间或热图 | 用于总体趋势和不确定性分析 |
| 单次代表运行 | 单seed代表案例 | 否 | 检查一条具体时间线、动作、轨迹和事件 | 只代表该seed，不能当成20-seed均值 |
| 双案例回放 | 阶段4C | 否 | 在同一绝对时间轴比较基准与压力代表案例 | 是single-seed对single-seed，不是统计显著性结论 |
| 场景背景 | 场景代理 | 可开关 | 绘制建筑、道路、农田、瓦砾、积水、丘陵等 | 明确标注“不参与实验计算” |
| 算法关系示意 | Non-IID演示 | 可调参数 | 解释FedAvg、FedProx、SCAFFOLD预期关系 | 不是课题实际训练日志 |
| 端侧实测 | benchmark.json | 用户导入 | 显示具体模型和设备的参数、延迟、内存 | 只有带完整测试条件的实测才可信 |

### 3.1 页面中的“—”代表什么

“—”通常表示当前数据源没有提供该字段，或者网站为避免混用旧数据而主动不显示。它不等于0。

例如：外部飞行日志没有汇总指标时，成功率显示“—”，不能解释为成功率为0%；没有`battery_wh`时，也不能把缺失值解释为电量耗尽。

## 4. 顶部导航、任务概览与数据模式

### 4.1 顶部场景切换

页面顶部中央有三个场景按钮：

- **物资配送**：城市楼宇投递与签收回传。
- **搜索救援**：灾区目标搜索、风险标注和间歇通信。
- **农田喷洒**：按作物风险分区执行定量喷洒。

切换场景会更新任务、地图背景、实时代理无人机和内置Python回放数据。

### 4.2 右上角状态时钟

右上角显示当前数据模式和`T + x.x s`：

- `T+`表示从当前任务轮次或回放起点开始经过的秒数。
- 亮起的状态点表示时间正在运行；暂停状态下时间不再前进。
- 切换回放、代表案例或场景时，时间可能返回起点。

### 4.3 任务概览区

任务标题右侧有四个概览指标：

| 指标 | 含义 | 高低如何理解 |
|---|---|---|
| 当前完成 | 已完成任务数 / 当前任务总数 | 越高表示当前轮次完成进度越大；必须结合时限判断是否“按时” |
| 平均电量 | 当前所有无人机电量百分比的平均值 | 通常越高储备越充足；低值可能触发返航，但高值也可能只是任务尚未执行 |
| 最高温度 | 当前机队中温度最高的一架 | 越接近或超过热安全上限，降载风险越高；不是平均温度 |
| 风速 | 实时代理当前风场速度 | 越高通常意味着飞控补偿和能耗压力更大；回放模式显示“—”，避免混入实时代理风场 |

### 4.4 数据模式按钮

| 按钮 | 内容 | 交互限制 |
|---|---|---|
| 实时代理 | 浏览器内状态机、风场、障碍和避碰 | 可以调整1–12架机群，也可以人工下发任务 |
| Python回放 | 原始`timeline.csv`事件和`summary.json`汇总 | 只读，保留原机数和任务归属 |
| 动力学回放 | 在Python事件之间平滑插值 | 只读，是平滑代理而非真实RotorPy/PyBullet结果 |
| 外部日志 | 用户选择标准JSON文件后回放 | 只读；文件没提供的指标保持缺失 |

实验代表案例不是顶部常驻按钮。它需要从实验工作台点击“在主地图回放此案例”后进入。

### 4.5 数据源摘要条

如果当前数据源带有可信汇总，右侧会显示：成功率、结果回传率、总能耗和约束违规数。

- **成功率高**：更多任务完成；仍需结合是否按时、能耗和安全约束。
- **回传率高**：更多已生成结果成功传回；任务完成率高而回传率低，说明“完成了作业但结果未及时回到地面端”。
- **能耗低**：在任务规模和场景相同的前提下通常更节能；不同任务量之间不能直接只比总能耗。
- **违规低**：通常越低越好，0表示汇总中没有记录硬约束违规。

## 5. 主地图：任务空间与三维航迹

主地图是网站最重要的交互区域。它把任务点、无人机、三维航迹、源事件和可选场景背景放在同一世界坐标视图中。

### 5.1 世界坐标与屏幕投影

- 地图内部保留数据源的`x / y / z`世界坐标。
- 缩放、平移、俯视或等距只改变屏幕显示，不改变任务坐标、遥测或安全距离。
- 数据明确声明单位为米时显示`m`和`m/s`。
- 外部日志未声明单位时，页面显示“源坐标单位未声明”，不会擅自标成米。

### 5.2 地图顶部的数据边界提示

地图上方有两行重要提示：

1. 当前任务、无人机、轨迹和事件来自哪个数据源。
2. 当前是否开启场景代理，以及场景代理“不参与计算”。

在解释地图前，建议先读这两行。尤其在实验回放中，实验真实几何和航迹始终优先；背景建筑不会改变自动适配范围，也不会改变实验轨迹。

### 5.3 地图中的主要对象

| 对象 | 画面含义 | 数据性质 |
|---|---|---|
| BASE基地 | 起飞、返航、补能或代理通信端点 | 可能来自任务定义；无基站坐标时明确标为代理端点 |
| 任务点 | 带任务编号的圆点 | 来自当前场景任务或实验任务定义 |
| 灰色任务点 | 等待执行 | 数据/任务状态 |
| 青色任务点 | 正在执行 | 数据/任务状态 |
| 绿色任务点 | 已完成 | 数据/任务状态 |
| 三维无人机 | 当前`x/y/z`位置、阴影、高度垂线和编号 | 来自当前数据模式 |
| 选中光环 | 当前被检查的无人机 | 交互状态，不是实验事件 |
| 实线/彩色航迹 | 无人机到当前时刻的三维历史轨迹 | 实时代理或数据源轨迹 |
| 菱形事件标记 | 返航、冷却、让行、禁飞绕行等关键动作 | 来自源时间线或实时代理 |
| 红色三维盒 | 带上下高度范围的禁飞区 | 实验数据源真实盒几何 |
| 红色虚线圆区 | 实时代理中的禁飞区 | 实时代理环境 |
| 建筑、道路、农田、瓦砾等 | 帮助识别城市、灾区、农田场景 | 场景代理，不参与实验计算 |

### 5.4 三维无人机模型如何读

- 无人机垂直投影到地面的阴影帮助判断水平位置。
- 高度垂线和`z`标签帮助判断飞行高度。
- 颜色区分不同无人机，并与机队列表、任务归属保持一致。
- `v`方向线只表示数据提供的速度方向，不等于完整姿态或真实机头朝向。
- 如果数据没有姿态角，网站不会伪造滚转、俯仰和偏航。

### 5.5 点击与拖动

- 点击地图上的无人机：选中该机，并同步更新机队遥测、决策链和安全边界。
- 拖动地图空白区域：平移视图。
- 地图获得键盘焦点后，使用方向键：按顺序切换选中无人机。

## 6. 地图图层、图例与专属数据层

地图标题右侧的图层按钮用于控制显示内容。按钮变灰且不可点，表示当前数据源没有提供该图层所需字段，并不表示字段值为0。

### 6.1 通用图层

| 图层 | 显示内容 | 关闭后会不会改变计算 |
|---|---|---|
| 三维无人机 | 三维机身、旋翼、阴影、高度和标签 | 不会，只隐藏模型；仍可用简化点表示 |
| 场景代理 | 建筑、道路、农田、瓦砾、积水、丘陵、林带等 | 不会；明确不参与实验计算 |
| 任务/航迹 | 任务点和历史航迹 | 不会，只改变显示 |
| 源事件 | 关键动作的空间标记 | 不会，只改变显示 |
| 禁飞区 | 实时代理禁飞区或数据源三维禁飞盒 | 关闭只隐藏；不改写源数据 |

场景代理旁边的“低/中/高”只控制绘制细节数量。细节变高不会增加实验障碍，也不会改变无人机轨迹。

### 6.2 实时代理专属图层

| 图层 | 显示内容 | 如何理解 |
|---|---|---|
| 复杂地形 | 高楼、瓦砾、丘陵、水塘、林带等安全环境 | 在实时代理中进入局部安全修正；和单纯场景代理背景不同 |
| 风场 | 风向箭头及当前风速 | 风越强，地速补偿和能耗压力通常越大 |
| 动态障碍 | 随时间变化的代理障碍 | 用于演示局部避障 |
| 机间距 | 相邻无人机的距离关系 | 红色表示低于阈值，黄色表示接近阈值，淡色表示较安全 |

### 6.3 通信图层

通信图层只在数据包含链路或缓存字段时可用。

- 青色链路：链路可用。
- 红色虚线和`LINK DOWN`：当前链路中断。
- `PDR`：分组投递率，接近1通常更好。
- `ms`：通信时延，场景相同且业务相同时通常越低越好。
- `CACHE x MB`：机载缓存已经占用的容量；持续升高说明结果在积压。
- “待上传结果”升高：更多结果尚未返回地面端。

高缓存并不必然表示任务失败。它可能说明“无人机继续完成任务并安全缓存结果”，但如果缓存继续增长并伴随丢弃数据增加，就说明通信压力已经影响结果闭环。

### 6.4 电量图层

电量图层按航迹采样点的电量为轨迹着色：

- 电量较高：偏绿色。
- 电量下降：逐渐向黄色、红色变化。
- 标注`RETURN`：已经达到或低于数据源定义的返航储备阈值。
- `TURNAROUND`：在基地补能、换载荷或准备下一架次。

电量高通常意味着储备更充足，但“任务结束时仍有高电量”也可能是任务较少或提前放弃，必须结合完成率判断。网站只有在数据提供了可信电池容量和储备参数时才按Wh解释；不会凭空绘制可达范围。

### 6.5 温度图层

温度图层按航迹采样点温度着色：

- 低温段偏蓝。
- 温度升高后向暖色过渡。
- 达到数据源热安全上限时标注`COOL_DOWN`。
- `LOCAL_UPDATE`表示发生轻量本地更新，可能带来额外计算与热负载。

温度越接近热上限，持续高功耗工作的余量越小。`COOL_DOWN`次数高表示安全门频繁工作，既说明热压力高，也说明保护机制正在触发，不能简单解释成算法失败。

### 6.6 多机冲突图层

多机冲突层使用三维距离，而不是只看平面投影：

- `d`：当前两机三维距离。
- `CPA`：按照当前相对速度预测的最近接距离；越低风险通常越大。
- `TTC`：预计到达安全距离阈值的剩余时间；正值越小表示需要越快处理，`—`表示当前趋势下没有预计进入阈值。
- 安全包络：无人机周围的虚线范围。
- 红线：已经低于安全阈值。
- 黄线：接近安全阈值。

当前距离高不代表未来一定安全，因此要结合CPA和TTC；当前距离不大但两机正在远离时，TTC可能为空。

### 6.7 三维禁飞盒

实验数据中的禁飞区以轴对齐三维盒显示，并标注`z_min–z_max`。它说明禁飞约束不仅有平面范围，还有高度上下界。

- 航迹从盒子平面外绕行：说明采用水平绕行。
- 航迹高度在盒子上方且满足上界：可能是垂直越障策略。
- 轨迹进入盒内：需要结合`no_fly_violations`判断是否记录为违规。

动态障碍如果数据包只提供数量、模型和避障事件，而没有逐时刻障碍坐标，网站不会伪造一条移动轨迹。

### 9.2 无人机卡片

每张无人机卡片包含：

| 内容 | 含义 |
|---|---|
| UAV编号和颜色 | 与地图无人机、航迹、任务归属一致 |

### 9.3 选中无人机详细遥测

| 字段 | 含义 | 高低如何理解 |
|---|---|---|
| 世界位置x/y | 水平位置 | 不是性能指标，只表示位置 |
| 世界高度z | 当前高度 | 用于判断高度层和垂直解冲突；不是越高越好 |
| 三维速度 | `sqrt(vx²+vy²+vz²)` | 高表示运动更快，但需结合安全、能耗和任务阶段 |
| 源事件时刻 | 当前回放帧对应的源事件时间 | 用于核对时间线 |
| 源电量Wh | 数据源记录的绝对剩余能量 | 越低越接近返航边界；需结合电池容量和储备阈值 |
| 剩余药量 | 农田实时代理的药液比例 | 越低表示已消耗越多，过低会限制后续喷洒 |
| 待回传结果 | 尚未成功上传的结果数量 | 通常越低越好；高值表示通信积压 |
| 源动作 | 数据源当前动作 | 说明当前决策语义 |
| 链路类型 | MAVLink、ROS 2或数据源声明 | 类型名称本身不表示质量 |
| 带宽 | 每秒可传输的数据量 | 在其他条件相同下通常越高越好 |
| PDR | 分组投递率 | 越接近1通常越可靠 |
| 时延 | 数据往返或传输延迟 | 在同类任务中通常越低越好 |
| 缓存 | 当前机载缓存占用 | 持续升高意味着回传受阻 |
| 避障模式 | 清晰、绕地形、让行、脱困 | 表示当前安全策略 |
| 停滞检测 | 未有效接近目标的累计时间 | 接近2.2秒时可能触发脱困侧移 |
| 模型版本 | 当前轻量任务头版本 | 版本高只表示发生过更新，不直接代表精度更高 |

### 9.4 四种实时避障状态

| 状态 | 含义 |
|---|---|
| 航迹清晰 | 当前没有需要强调的局部安全修正 |

## 10. 决策链与安全边界

选中无人机后，主地图下方会出现“决策链”和“数学边界”两个区域。

### 10.1 决策链动作解释

| 动作 | 中文含义 | 表示什么 |
|---|---|---|
| ASSIGN | 接收与校验任务 | 检查电量、载荷、时限和安全可行性 |
| FLY_AND_INFER | 飞行与机载推理 | 沿航迹飞行，并持续执行感知和风险判断 |
| SERVICE | 现场任务执行 | 投递、搜索或喷洒，并生成任务结果 |
| COMMUNICATE | 通信与结果回传 | 链路可用时上传结果 |
| STORE_AND_WAIT | 缓存并等待链路 | 作业结果已生成，但通信暂不可用 |
| DEFER_UPLOAD | 延迟结果上传 | 链路容量不足，延后上传 |
| LOCAL_UPDATE | 轻量本地更新 | 更新小型任务头，不训练大模型 |
| COOL_DOWN | 热安全降载 | 达到热安全门，暂停高功耗活动并冷却 |
| YIELD | 确定性机间让行 | 根据通行权减速并侧向绕行，避免对称卡死 |
| DECONFLICT_CLIMB | 垂直解冲突 | 改变真实高度，打破平面冲突 |
| AVOID_DYNAMIC | 动态障碍避让 | 局部安全门修正航迹 |
| AVOID_NO_FLY | 禁飞区绕行 | 根据三维禁飞盒执行全局绕行 |
| RETURN | 安全返航 | 在储备电量边界内返回基地 |
| TURNAROUND | 补能与周转 | 更换电池、补充载荷或药液 |
| HOLD | 等待可行任务 | 当前没有满足条件的任务，安全待命 |

在实时代理中，决策链显示网站当前状态机阶段和触发条件；在回放中，它显示数据源动作，不会用实时代理逻辑覆盖源事件。

### 10.2 安全边界四项

| 边界 | 页面含义 | 如何判断 |
|---|---|---|
| E：储备电量 | 实时代理要求`Eᵢ(t) ≥ 18%` | 低于边界触发返航；回放显示源Wh值 |
| T：热安全 | 实时代理要求`Tᵢ(t) ≤ 68°C` | 达到边界触发冷却降载 |
| D：地形、障碍和机间距 | 实时代理机间安全距离为5m，并避开环境影响半径 | 距离低于阈值表示风险门激活 |
| W：风场补偿 | `v_ground = v_cmd + w(t)` | 风越强，控制补偿和能耗影响通常越大 |

“ALL SAFE”表示当前显示的实时代理守门条件都满足；“GATE ACTIVE”表示至少一个边界正在触发保护动作。回放模式显示“SOURCE DATA”，说明这里只读展示源字段。

## 11. 当前数据源汇总

该区域汇总当前Python结果或实验代表案例。外部日志没有汇总时会显示“NO SUMMARY”，不会挪用其他场景的旧数字。

| 指标 | 含义 | 一般方向 |
|---|---|---|
| 按时完成 | 截止时间内完成数 / 任务总数 | 在任务定义相同时越高越好 |
| 成功率 | 完成任务占比 | 越高通常越好 |
| 结果回传率 | 成功返回地面端的结果占比 | 越高通常越好 |
| 丢弃数据 | 因链路、缓存或策略无法保留的数据量 | 越低通常越好 |
| 总能耗 | 当前运行所有无人机累计能耗 | 相同任务量下越低越节能；任务量不同不能直接比较 |
| 最小间距 | 整个运行中记录的最小三维机间距 | 高于安全阈值才具有安全意义；并非无限越高越好 |
| 违规 | 数据源记录的硬约束违规事件数 | 通常越低越好，0最好 |

### 11.1 不要只看单个指标

典型组合解释：

- 完成率高、回传率低：作业完成了，但通信闭环较差。
- 完成率高、能耗高：任务收益好，但资源成本较高。
- 完成率下降、违规为0：安全策略可能主动放弃部分任务以守住边界。
- 最小间距高、任务时长也高：可能通过更保守的避让换取安全。

## 12. 端侧模型真实基准接口

“EDGE MODEL / 真实基准接口”只在导入真实`benchmark.json`后显示具体数字。

| 指标 | 含义 | 高低如何理解 |
|---|---|---|
| 参数量 | 模型可训练参数数量，单位M（百万） | 在任务能力相近时越低通常越适合端侧；参数少不自动等于精度高 |
| 基线占比 | 当前模型参数量 / 显式基线参数量 | 小于10%表示满足页面设定的轻量化目标 |
| 中位推理延迟 | 多次推理耗时的中位数 | 在相同设备、输入、精度和运行库下越低越快 |
| 峰值内存 | 进程运行期间峰值内存占用 | 在相同条件下越低越适合受限设备；页面以100MB作为显示边界 |

### 12.1 为什么不同设备的数字不能直接混比

峰值内存和延迟同时受以下条件影响：

- 模型文件和网络结构；
- 输入尺寸；
- FP32、FP16或INT8精度；
- ONNX Runtime、TensorRT等运行库；
- CPU、GPU、NPU和驱动；
- 线程数、预热次数和测试轮数。

因此，只有测试条件一致的结果才适合横向比较。默认“—”表示尚无实测，不表示性能为0。

## 15. 如何阅读均值曲线、95%置信区间和热图

### 15.1 均值曲线

- 横轴：实验主动改变的条件，例如中断率、无人机数量或安全距离。
- 纵轴：20个seed的指标均值。
- 实线：均值趋势。
- 圆点：一个具体参数组合的均值。
- 不同颜色：不同场景或分组系列。

曲线上升或下降本身没有统一的好坏，必须先判断纵轴是什么。例如完成率上升通常是好现象，而能耗、违规或丢弃数据上升通常是成本增加。

### 15.2 95%置信区间阴影

- 阴影窄：不同seed结果更集中，均值估计更稳定。
- 阴影宽：不同seed差异更大，均值不确定性更高。
- 两组阴影明显分开：提示差异可能较稳定，但仍不等同于完成了显著性检验。
- 两组阴影重叠：不能直接下结论说“没有差异”；需要原始样本和正式统计检验。

### 15.3 热图

- 横轴和纵轴共同表示两个实验条件。
- 每个格子内的数字是该配置的20-seed均值。
- 本网站热图从较暗的蓝绿向更亮、更暖的颜色过渡：较暖、较亮通常代表数值较高，较暗、较冷通常代表数值较低。
- 颜色只表示数值大小，不自动表示好坏。
- 白色边框表示当前选中的格子。

例如，“完成率”热图中暖色高值通常更好；“能耗”热图中暖色高值表示成本更高；“COOL_DOWN次数”暖色表示热安全门触发更频繁。

### 15.4 常见指标方向

| 指标 | 高值通常表示 | 一般评价方向 |
|---|---|---|
| 任务完成率、按时完成率 | 更多任务成功或按时完成 | 通常越高越好 |
| 结果回传率、PDR | 通信结果更完整可靠 | 通常越高越好 |
| 任务吞吐 | 单位时间完成更多任务 | 通常越高越好 |
| 最小机间距 | 最危险时刻仍保持更大距离 | 高于安全阈值更好，但无需无限增大 |
| 丢弃数据、违规 | 数据损失或安全违规更多 | 通常越低越好 |
| 总能耗、总航程、任务时长 | 资源或时间成本更高 | 同等任务下通常越低越好 |
| 缓存事件 | 链路中断时采用缓存的次数 | 高值表示通信压力或保护机制更常触发，不能单独判好坏 |
| COOL_DOWN | 热安全降载次数 | 高值表示热压力大，同时说明安全门在工作 |
| YIELD、垂直解冲突、动态避障 | 安全策略触发次数 | 高值表示冲突/障碍压力或保护动作更多，不等于违规 |
| 基地周转 | 返回基地补能/换载荷次数 | 高值表示周转需求和保障压力更大，也可能支撑了更多任务 |
| 本地更新 | 轻量任务头更新次数 | 高值只表示更新更频繁，不直接代表模型质量更高 |

## 16. 五组实验图表逐项分析

### 16.1 实验01：通信中断持续作业

实验目的：区分“任务已经完成”和“结果已经回传”，识别链路、缓存和数据降级边界。

| 图表 | 横轴增大代表 | 纵轴高/低代表 | 建议观察 |
|---|---|---|---|
| 中断率—结果回传率 | 断链概率更高 | 回传率高较好，低表示更多结果未返回 | 曲线何时开始明显下降，以及置信区间是否扩大 |
| 连续中断—缓存事件 | 单次连续断链更长 | 高表示更频繁使用机载缓存 | 若缓存事件升高但丢弃仍低，说明缓存有效；若两者都升高，说明容量受压 |
| 带宽—丢弃数据 | 可用带宽更高 | 丢弃越低越好 | 是否存在带宽达到某值后丢弃趋近稳定的容量门槛 |
| PDR—丢弃数据 | 分组投递更可靠 | 丢弃越低越好 | PDR下降时数据损失增加的敏感区间 |
| 时延—丢弃数据 | 通信时延更大 | 丢弃越高越差 | 高时延是否占用上传时隙并导致降级 |
| 存储—丢弃数据 | 机载缓存容量更大 | 丢弃越低越好 | 丢弃不再明显下降的位置可作为容量饱和参考 |

不能只用任务完成率评价这一实验：无人机可能完成现场任务，却因通信中断没有把结果传回。

### 16.2 实验02：电量与安全返航

实验目的：分析初始电量、返航储备、截止时间和任务负载之间的安全收益折中。

| 图表 | 横/纵轴含义 | 高/低如何理解 | 建议观察 |
|---|---|---|---|
| 初始电量×储备—完成率 | 横轴为初始Wh，纵轴为预留返航Wh | 完成率高表示完成更多；储备提高后完成率下降可能是主动保守，而非算法失效 | 找到既无安全违规又能保持合理完成率的区域 |
| 截止时间—完成率 | 横轴越大通常表示截止时间更宽松 | 完成率越高越好 | 哪类场景对时限收紧最敏感 |
| 任务倍数—完成率 | 横轴越大表示任务负载越重 | 高负载下仍保持高完成率表示承载能力较强 | 曲线拐点表示容量接近上限 |
| 任务倍数—能耗 | 任务量增加 | 总能耗高表示累计资源成本高 | 任务更多时能耗上升是正常现象，不能脱离完成任务数判断效率 |
| 任务倍数—基地周转 | 任务量增加 | 周转高表示补能/换载荷更频繁 | 结合完成率判断周转是否有效支撑了持续作业 |

严格储备策略可能降低完成率，却减少低电量风险。评价时应同时看最低电量、返航事件和违规数。

### 16.3 实验03：温度与计算负载

实验目的：研究环境温度、持续推理负载和本地更新对热安全门的影响。

| 图表 | 横/纵轴含义 | 高/低如何理解 | 建议观察 |
|---|---|---|---|
| 温度×负载—COOL_DOWN | 环境更热、计算负载更高 | 暖色高值表示冷却降载更频繁 | 热安全门从哪个温度/负载组合开始频繁触发 |
| 温度×负载—按时完成率 | 环境更热、计算负载更高 | 完成率高较好，低表示热降载影响截止时间 | 与COOL_DOWN热图一起看，判断保护代价 |
| 热上限—按时完成率 | 热安全上限变宽或变严 | 高完成率表示任务收益高 | 提高热上限可能提高完成率，但不代表硬件更安全 |
| 热上限—COOL_DOWN | 热安全上限变化 | 高值表示保护动作频繁 | 上限过严可能频繁降载；过松可能减少降载但提高温度风险 |
| 更新策略—本地更新 | 关闭、基准或高负载更新策略 | 高值表示更新次数多，不等于模型一定更好 | 结合最高温度和按时完成率判断更新成本 |

`COOL_DOWN=0`不一定总是好：它可能表示环境确实安全，也可能表示热上限设置过松。需要同时查看最高温度和违规。

### 16.4 实验04：多机三维解冲突

实验目的：评估机群规模增加带来的并行收益，以及让行和垂直解冲突的代价。

| 图表 | 横/纵轴含义 | 高/低如何理解 | 建议观察 |
|---|---|---|---|
| 机群规模—完成率 | 无人机数量增加 | 完成率高较好 | 固定24任务下，多加无人机是否继续带来收益 |
| 机群规模—任务时长 | 无人机数量增加 | 时长低通常更好 | 曲线趋平说明并行收益开始饱和 |
| 机群规模—吞吐 | 无人机数量增加 | 吞吐高通常更好 | 是否出现规模增加但吞吐不再明显增长 |
| 机群规模—避让 | 无人机数量增加 | 高值表示冲突处理更多，不是违规数 | 判断规模收益是否被避让开销抵消 |
| 规模×安全距离—避让 | 两个条件共同增加避让压力 | 暖色高值表示更多避让 | 找到机数和安全距离共同造成高压力的区域 |
| 迎面场景—YIELD | 安全距离变化 | 高值表示确定性侧向让行更常触发 | 应结合最小间距和是否卡死判断有效性 |
| 迎面场景—垂直解冲突 | 安全距离变化 | 高值表示更多使用高度层打破对称冲突 | 检查真实z变化和最小三维间距 |

避让事件少并不自动表示更好：可能是冲突少，也可能是安全机制没有触发。应至少同时看完成率、最小间距、违规数和任务时长。

### 16.5 实验05：禁飞区与动态障碍

实验目的：分离三维禁飞盒造成的全局绕行和动态障碍造成的局部避让。

| 图表 | 横/纵轴含义 | 暖色高值代表 | 建议观察 |
|---|---|---|---|
| 禁飞区×障碍—航程 | 横轴为动态障碍数量，纵轴为禁飞区复杂度 | 总航程更长、全局绕行成本更高 | 禁飞区复杂度和动态障碍是否产生叠加代价 |
| 禁飞区×障碍—能耗 | 同上 | 总能耗更高 | 航程增加是否同步带来能耗增加 |
| 禁飞区×障碍—动态避障 | 同上 | 局部避障事件更多 | 动态障碍数量增加后保护动作是否线性或突增 |
| 禁飞区×障碍—完成率 | 同上 | 完成率更高，通常更好 | 障碍压力在哪些组合下造成超时或任务放弃 |

禁飞区违规应尽量为0。绕行事件和航程增加可以是遵守安全约束所付出的合理代价。

## 18. 阶段 4C：基准与压力双地图同步对照

阶段4C用两个single-seed代表案例做同步回放，主要用于检查事件顺序、航迹差异和机制触发。

### 18.1 五类案例对

| 标签 | A基准 | B压力/机制 | 重点观察 |
|---|---|---|---|
| 实验01·通信 | 通信基准 | 链路压力 | 断链、缓存、延迟上传和回传率 |
| 实验02·电量 | 配送基准 | 严格储备 | 返航触发、能耗和完成率 |
| 实验03·温度 | 搜救基准 | 高温高负载 | 温升、COOL_DOWN和按时完成 |
| 实验04·多机 | 三机 | 八机 | 最小间距、让行、垂直解冲突和吞吐 |
| 实验05·禁飞 | 无屏障基准 | 双禁飞盒与五障碍组合压力 | 三维绕行、动态避障和任务代价 |

### 18.2 共享控制

- **联动视角**：只有场景和任务几何可比时才可开启；一侧缩放、平移或切换投影会同步到另一侧。
- **幽灵航迹**：只有场景、米制坐标、任务几何和无人机数量全部一致时才可开启。
- **场景代理**：仅切换非计算背景。
- **三维无人机**：切换当前三维模型显示。
- **源事件**：切换时间线关键事件空间标记。

三机与八机案例机数不同，因此幽灵航迹被强制禁用，避免把不同编号、不同规模的轨迹错误配对。

### 18.3 幽灵航迹是什么

开启后，一侧地图用虚线显示另一侧同编号无人机截至当前时刻的轨迹，便于观察空间偏差。

幽灵航迹不是预测轨迹，也不是20-seed平均轨迹。它只是另一个可比single-seed案例的实际源轨迹。

### 18.4 双地图时间同步

- 两侧共享一个绝对秒时间轴。
- 播放、暂停、回到起点和关键事件跳转同时作用于两侧。
- 两个案例时长不同时，较短案例到达末尾后冻结最后一帧，不循环、不外推。
- 每张地图仍保留自己的缩放、适配、重置和投影控制。
- 窄屏下使用“A基准/B压力”按钮切换地图，但仍共享同一时间轴。

### 18.5 共享机队检查器

页面只有一个共享检查器：

1. 先选择A基准或B压力。
2. 再选择无人机编号。
3. 查看该机当前动作、源事件时间、世界坐标、三维速度、电量、温度、链路、缓存和待上传结果。

“源事件时间”是最近源采样的时刻；“当前主时间”是共享时间轴位置，两者可能因采样间隔略有不同。

### 18.6 B−A代表案例差值

差值区只比较当前两个single-seed代表运行：

| 指标类型 | B−A为正通常表示 |
|---|---|
| 完成率、回传率、吞吐 | B在该次运行中更高 |
| 最小三维间距 | B在该次运行中的最危险间距更大 |
| 能耗、航程、时长、温度 | B成本或压力更高 |
| 丢弃数据、违规 | B数据损失或安全问题更多 |
| 返航、冷却、让行、避障次数 | B中对应机制触发更多；不能简单等同于更差 |

这些差值不带20-seed置信区间，不能替代统计推断。若要判断总体趋势，应返回实验工作台查看均值和95%置信区间。

## 19. 任务队列表

任务队列按行展示当前场景或实验代表案例的任务。

| 列 | 含义 |
|---|---|
| 任务 | 唯一任务编号，如L1、R3、S6 |
| 地点与原因 | 真实语义化地点、任务内容、载荷/风险信息和必须前往的原因 |
| 优先级 | 亮点越多表示优先级越高 |
| 截止 | 相对任务起点的截止时间`T+x s` |

### 19.1 优先级和截止时间不是一回事

- 优先级高：调度时更重要。
- 截止时间短：时间更紧迫。
- 实时代理会综合优先级、时限、电量和安全可行性，而不是只按一个字段排序。

### 19.2 地点和任务原因

地点名称和原因用于解释任务语义，例如“和安社区卫生站—慢病患者急需处方药”或“老城商场废墟—高温目标疑似被困人员”。它们帮助观看者理解无人机为何前往该区域，不是额外的地理定位服务。

网站支持导入标准JSON回放。基本结构为：

{
  "engine": "RotorPy",

### 20.1 导入后保留什么

## 21. 典型使用流程

### 21.1 面向普通参观者：快速理解网站

3. 点击地图上的无人机。

2. 切换“Python回放”。
3. 暂停并拖动时间轴。
4. 使用动作筛选或“下个关键”。

### 21.3 面向实验分析：统计趋势到具体案例

1. 在实验工作台选择实验。
2. 阅读实验目的和统计口径。
3. 选择图形，先看总体趋势和置信区间。

4. 点击“导入benchmark.json”。
5. 只与相同测试条件的结果比较。

## 22. 常见误读与注意事项

### 22.1 “场景里画了建筑，所以实验无人机在避建筑”

不一定。实验模式中的建筑、道路、农田等是场景代理，并明确标注“不参与计算”。只有数据源真实任务、轨迹、事件和禁飞盒才能作为实验内容解释。

### 22.2 “20-seed均值可以画成一条平均航迹”

不能。不同seed的事件时刻、任务顺序和空间路径未必一一对应。网站只把20-seed用于标量统计，不生成不存在的平均轨迹。

### 22.3 “代表案例完成率高，说明算法总体更好”

不能这样推断。一个seed只是一个运行。需要查看20-seed均值、置信区间，并在必要时做正式统计检验。

### 22.4 “避让次数越多越差”

不一定。避让多可能说明环境压力高，也可能说明安全机制正常工作。必须结合违规、最小间距、任务时长和完成率。

### 22.5 “能耗越低越好”

只有任务量、场景和完成程度相同或可比时才成立。少做任务自然可能能耗更低。

### 22.6 “电量百分比和Wh可以直接混用”

不能。百分比是相对容量，Wh是绝对能量。不同电池容量下，相同百分比不一定代表相同可用能量。

### 22.7 “最高温度低说明性能一定好”

不一定。低温可能来自有效热管理，也可能来自减少推理、提前返航或任务未执行。应结合完成率和计算事件。

### 22.8 “外部日志没有某项指标，所以该指标是0”

不是。缺失显示“—”，表示未知。网站刻意不使用其他数据源填补。

### 22.9 “动力学回放是真实物理仿真”

不是。它是二阶平滑代理。真实RotorPy、Gym-PyBullet-Drones或飞控结果需要转换后从“外部日志”导入。

### 22.10 “这个网站可以控制真实无人机”

不能。网站仅用于科研解释、仿真展示、日志审查和实验对照。

## 23. 术语速查

| 术语 | 解释 |
|---|---|
| seed | 仿真随机种子；固定seed用于复现实验 |
| single-seed | 一次具体运行，而不是多次运行均值 |
| 20-seed统计 | 对20个固定seed进行的均值和置信区间统计 |
| 95% CI | 95%置信区间；反映均值估计的不确定性 |
| PDR | Packet Delivery Ratio，分组投递率 |
| CPA | Closest Point of Approach，按当前相对运动预测的最近接距离 |
| TTC | Time to Constraint/Collision，预计进入安全阈值的时间 |
| Non-IID | 各无人机本地数据分布不一致 |
| FedAvg | 基础联邦平均算法 |
| FedProx | 通过近端项缓解本地训练漂移 |
| SCAFFOLD | 通过控制变量校正客户端漂移 |
| 场景代理 | 解释性三维背景；不参与实验计算 |
| 源事件 | 数据源时间线中记录的动作事件 |
| 返航储备 | 为安全返回基地预留的能量 |
| 周转 | 在基地补能、换载荷或补充药液后再次出动 |
| 幽灵航迹 | 另一个可比single-seed案例的真实源轨迹叠加，不是预测或均值 |

## 最后建议

阅读网站时，始终按以下顺序提问：

1. **当前是什么数据模式？**
2. **这个对象来自数据源，还是场景代理？**
3. **图中是20-seed统计，还是一个single-seed案例？**
4. **这个指标越高究竟代表收益、成本，还是安全机制触发？**
5. **结论是否同时考虑了完成、通信、能量、温度和安全？**

只要先分清数据来源，再结合多个指标和时间线阅读，就能正确理解网站每个功能区所表达的内容。
